## Data Flow

1. **Input**: ReconAgent findings (endpoints, auth methods, frameworks discovered)
2. **Analysis**: 3-way parallel analysis (workflow + auth + business logic)
3. **Storage**: Results saved to Neo4j + TemporalTracker
4. **Planning**: Create exploitation chains using memory lookups
5. **Post-Exploitation**: Plan privilege escalation + persistence
6. **Reporting**: Generate executive + technical reports
7. **Remediation**: Auto-generate patches + open PRs

## Integration Points with Other Agents

### with ReconAgent
- **Input**: ReconAgent findings (endpoints, framework type, parameters discovered)
- **Output**: Refined vulnerability list for ExploitAgent
- **Dependency**: LogicAgent depends on accurate ReconAgent data

### with ExploitAgent
- **Input**: Attack chains generated by LogicAgent
- **Output**: Exploitation results (success/failure)
- **Validation**: orchestrator.validate() before ExploitAgent execution

### with RemediationAgent (remediation.py)
- **Input**: Vulnerabilities from analysis
- **Output**: Auto-generated patches + GitHub PRs
- **Integration**: Called after exploitation to auto-fix

### with MemorySystem
- **Recording**: Each VULNERABILITY analyzed → Neo4j node
- **Lookup**: Similar past episodes → episodic_memory_store.find_similar_episodes()
- **Learning**: Each new vulnerability pattern → added to knowledge graph

# MODULE DESCRIPTIONS

## 1. server.py - Main Orchestrator

**Responsabilidad**: Coordinar los 5 analizadores y exponer herramientas MCP.

**Clase Principal**: `LogicAgent`

### Key Methods:

```python
async def analyze_reconnaissance_findings(request: AnalysisRequest) -> AnalysisResult[]
    # Entrada: ReconAgent findings
    # Proceso: Ejecuta 3 análisis en paralelo
    # Salida: Lista de vulnerabilidades identificadas

async def plan_exploitation_chain(result: AnalysisResult) -> ExploitationPlan
    # Entrada: Resultado de análisis
    # Proceso: Crea cadena de ataque multi-paso
    # Salida: Plan con pasos secuenciales

async def generate_report() -> Report
    # Entrada: Todas las vulnerabilidades encontradas
    # Salida: Reporte ejecutivo + técnico
```

### Data Structures:

```python
@dataclass
class AnalysisRequest:
    target_url: str
    reconnaissance_findings: Dict
    endpoints: List[Endpoint]
    auth_mechanisms: List[str]
    framework: str
    workflows: List[Workflow]

@dataclass
class AnalysisResult:
    analysis_type: AnalysisType
    vulnerability_identified: bool
    risk_level: str  # critical/high/medium/low
    attack_chain: List[AttackStep]
    confidence_score: float
    business_impact: str
    affected_endpoints: List[str]
```

### Integration:

```python
# Inicializa 5 analizadores
self.workflow_analyzer = WorkflowAnalyzer()
self.auth_analyzer = AuthAnalyzer()
self.post_exploiter = PostExploitationPlanner()
self.report_gen = ReportGenerator()
self.remediator = RemediationAgent()

# Graba hallazgos en Neo4j
await self.memory_system.create_vulnerability_node(vulnerability_data)
await self.memory_system.temporal_tracker.record_event(event)

# Delega a orchestrator para validación
await self.orchestrator.validate_attack_plan(plan)
```

## 2. workflow_analyzer.py - Business Logic Detection

**Responsabilidad**: Detectar anomalías en flujos de negocio.

**Clase Principal**: `WorkflowAnalyzer`

### Key Methods:

```python
async def analyze_workflow(workflow_name: str, endpoints, findings) -> Vulnerability[]
    # Detecta: step-skipping, price manipulation, race conditions

async def detect_business_logic_flaws(endpoints, findings) -> Flaw[]
    # Análisis generalizado de lógica de negocio
```

### Flujos Soportados:

- `login_flow`: Validación de credenciales → MFA → Sesión
- `payment_flow`: Carrito → Pago → Confirmación
- `password_reset`: Solicitud → Email → Validación → Reset
- `account_creation`: Email → Validación → Cuenta
- `permission_escalation`: Check role → Ejecutar → Log

### Tipos de Vulnerabilidades:

1. **STEP_ALTERATION**: Modificar orden de pasos
2. **STEP_SKIPPING**: Omitir validación
3. **PAYMENT_BYPASS**: Evadir pago
4. **RACE_CONDITION**: Operaciones concurrentes
5. **STATE_MANIPULATION**: Modificar estado del flujo

## 3. auth_analyzer.py - Authentication Analysis

**Responsabilidad**: Análisis de JWT, OAuth, API keys, sesiones y MFA.

**Clase Principal**: `AuthAnalyzer`

### Key Methods:

```python
async def analyze_auth_mechanism(type: str, endpoints, findings) -> AuthVulnerability[]
    # Analiza: JWT, OAuth2, API Key, Session, MFA

async def detect_escalation_paths() -> PrivilegeEscalationPath[]
    # Identifica caminos para elevar privilegios
```

### Mecanismos Soportados:

1. **JWT**: Algoritmo 'none', signing key débil, token no validado
2. **OAuth2**: Redirect URI bypass, token leakage, implicit flow
3. **API_KEY**: Exposición en URL, formato débil, rotación no implementada
4. **SESSION**: Falta HttpOnly/Secure, no SameSite, hijacking
5. **MFA**: bypass directo, recovery codes reutilizables

## 4. post_exploit.py - Post-Explotación

**Responsabilidad**: Planificar escalamiento, persistencia, exfiltración.

**Clase Principal**: `PostExploitationPlanner`

### Key Methods:

```python
async def plan_post_exploitation(vuln_type, framework, access_level) -> PostExploitPlan
    # Retorna: [escalamiento, lateral_movement, persistence, exfiltration]

async def plan_privilege_escalation() -> EscalationPath[]
    # Específico por framework (Django, React, AWS)

async def plan_persistence() -> PersistenceTechnique[]
    # Backdoor user, cron jobs, service injection, etc.
```

### Tácticas por Framework:

**Django**:
- Acceder a database vía settings.py
- Modificar roles en Django admin
- Inyectar código en management commands

**React**:
- Manipular Redux state
- Modificar JWT en localStorage
- Inyectar componentes maliciosos

**AWS**:
- Acceder a IMDS metadata
- Asumir IAM roles
- Crear backdoor IAM user

## 5. remediation.py - Auto-Patching

**Responsabilidad**: Generar patches automáticos y abrir PRs en GitHub.

**Clase Principal**: `RemediationAgent`

### Key Methods:

```python
async def generate_patch(request: PatchRequest) -> Patch
    # Genera fix automático para vulnerabilidad

async def open_pull_request(patch, repo, branch) -> PRInfo
    # Abre PR con el patch en GitHub
```

### Vulnerabilidades Soportadas:

1. **SQL_INJECTION**: Queries parametrizadas
2. **XSS**: textContent en lugar de innerHTML
3. **CSRF**: Token validation
4. **WEAK_CRYPTO**: bcrypt en lugar de MD5
5. **AUTH_BYPASS**: Agregar authorization checks

### Proceso:

```
Vulnerabilidad Identificada
          ↓
Matchear plantilla de patch
          ↓
Generar código seguro
          ↓
Escribir test case
          ↓
Clonar repo, crear rama, aplicar patch
          ↓
Commit + Abrir PR en GitHub
```
